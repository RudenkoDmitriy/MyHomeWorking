Здравствуйте.

Проект, в котором я участвую, назвается RoboCP. Суть проекта - создание системы управления летательным аппаратом ArduCopter. Конечный итог нашего проекта должен быть таким : с помощью GUI пользователь в реальном времени получает видеопоток с робота и может им управлять. Кроме того, робот должен уметь сам выявлять отклонения от намеченного курса и корректировать свое движение относительно их.

Мои задачи в данном проекте:
 -научить собирать проект с помощью системы CMake;
 -переписать код обработки Config'а;
 -избавиться в проекте от использования Boost и перейти на QT;
 -разобраться с разделом компьютерного зрения Optical flow, который реализован в библиотеке OpenCV, и научиться с помощью него определять характер движения робота.

Для чего все это нужно:
 -Наш проект собирается с помощью системы CMake, а не Ctrl-F5(как в общем-то многие крупные проекты);
 -Config в проекте представлен линейным списком, что весьма нечитабельно и сложно для дальнейшей разработки. Кроме того, внутри кода создается только один объект настроек(объект класса Config), который передается во все модули проекта. Зачем камере знать о настройках кинекта? Соотвественно, мной должна быть реализована новая иерархия Config'ов, где каждому модулю передавались бы собственные настройки;
 -QT шире и приятней в использовании, чем boost(было обсуждено в команде). Кроме того, если код переписать под использование только одной бибилиотеки, пользователю придется меньше ставить дополнительного ПО(или скачивать файл меньшего объема);
 -Если робота запускать внутри помещения, относительно данных акселерометра ему все-равно будет сложно не столкнуться с каким-то объектом(стол, шкаф, стена). Соответственно, хотелось бы кроме акселерометра иметь еще один источник данных о характере движения робота.
